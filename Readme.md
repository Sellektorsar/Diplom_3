# Тестирование веб-приложения Stellar Burgers

Автоматизированное тестирование веб-приложения Stellar Burgers с использованием паттерна Page Object Model.

## Структура проекта

```
├── locators/             # Локаторы элементов
│   └── locators.py       # Все локаторы приложения
├── pages/                # Page Object классы
│   ├── base_page.py      # Базовый класс для всех страниц
│   ├── main_page.py      # Главная страница
│   ├── login_page.py     # Страница входа
│   ├── register_page.py  # Страница регистрации
│   ├── profile_page.py   # Профиль пользователя
│   ├── order_feed_page.py # Лента заказов
│   ├── forgot_password_page.py # Восстановление пароля
│   └── reset_password_page.py  # Сброс пароля
├── tests/                # Тестовые файлы
│   ├── test_main_functionality.py # Тесты основного функционала
│   ├── test_order_feed.py      # Тесты ленты заказов
│   ├── test_password_reset.py  # Тесты восстановления пароля
│   └── test_profile.py         # Тесты профиля
├── utils/                # Вспомогательные утилиты
│   └── api_client.py     # Клиент для API запросов
├── conftest.py           # Фикстуры pytest
├── pytest.ini            # Настройки pytest
└── requirements.txt      # Зависимости проекта
```

## Установка и настройка

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Установка Allure

Для генерации отчетов необходимо установить Allure:

**Windows:**
```bash
scoop install allure
```

**macOS:**
```bash
brew install allure
```

**Linux:**
```bash
sudo apt-add-repository ppa:qameta/allure
sudo apt-get update
sudo apt-get install allure
```

## Запуск тестов

### Запуск всех тестов в Chrome

```bash
pytest --browser=chrome
```

### Запуск всех тестов в Firefox

```bash
pytest --browser=firefox
```

### Запуск конкретного тестового файла

```bash
pytest tests/test_password_reset.py --browser=chrome
```

### Запуск с подробным выводом

```bash
pytest --browser=chrome -v
```

## Генерация Allure отчета

### Запуск тестов с генерацией результатов

```bash
pytest --browser=chrome --alluredir=allure-results
```

### Просмотр отчета

```bash
allure serve allure-results
```

## Особенности реализации

### Page Object Model

- **Базовый класс** (`BasePage`): содержит общие методы для работы с элементами
- **Отдельный класс для каждой страницы**: обеспечивает модульность и удобство сопровождения
- **Локаторы в отдельном файле**: централизованное управление селекторами элементов
- **Использование expected_conditions**: все ожидания реализованы через явные ожидания Selenium

### Фикстуры

- **test_user**: автоматически создает тестового пользователя через API
- **test_ingredients**: получает актуальный список ингредиентов
- **driver**: настраивает WebDriver для выбранного браузера (Chrome/Firefox)
- **Автоматическое удаление пользователей**: после тестов пользователи удаляются через API

### API интеграция

- Создание пользователей через API
- Авторизация и получение токенов
- Создание тестовых заказов
- Очистка тестовых данных

## Кроссбраузерное тестирование

Все тесты запускаются в двух браузерах:
- **Chrome**: 17/17 тестов проходят успешно
- **Firefox**: 16/17 тестов проходят успешно

**Примечание:** Один тест (увеличение счетчика ингредиента) может не проходить в Firefox из-за особенностей реализации drag-and-drop в этом браузере.

- Python 3.8+

## Технологии

- **pytest** - фреймворк для тестирования
- **Selenium WebDriver** - автоматизация браузера
- **Allure** - генерация отчетов
- **requests** - работа с API
- **Page Object Model** - паттерн проектирования
